# 🌱 Cultivating the Dojo: Contributing to Dojo Ascension

Welcome, fellow developer, learner, and cultivator of code. 

**Dojo Ascension v4.0** is an interactive terminal RPG that teaches Python, Git, and JSON. Our mission is rooted in a solarpunk ethos: to democratize the technical skills required for navigating modern AI systems like Outlier's contributor skill assessments. Technology should not be a walled garden, but a thriving, open ecosystem.

Whether you are an Initiate just grasping data types or an Expert mastering Object-Oriented Programming, your contributions nourish this project. 

## 🌿 How You Can Contribute

We welcome contributions of all sizes! Here are the primary ways you can help the Dojo grow:

### 1. Expanding the Curriculum
The Dojo currently features 10 core missions ranging from System Grounding to Code Review Mastery. The `CURRICULUM` and `MISSIONS` dictionaries are designed for easy extension. 
* **Propose a Mission:** Open an issue outlining a new mission concept. What Outlier skill does it teach? What is the narrative?
* **Write a Mission:** Add a new entry to the dictionaries and draft the `mission_N_name(player)` function.

### 2. Tending to Bugs and Typos
Even the most well-tended gardens have weeds. If the terminal evaluates an answer incorrectly or you spot a typo in the narrative, please open an issue or submit a pull request fixing the logic.

### 3. Enhancing the VSCode Integration
We pair our missions with official VSCode tutorials and generate practice `.json` and `.py` files. If you have ideas for better IDE integration or practice files, let us know!

## 🛠️ Contribution Workflow

1. **Fork the Repository:** Create your own branch of the Dojo.
2. **Set Up Your Environment:** Create a virtual environment (`python -m venv venv`) and install our required dependencies.
3. **Branch Out:** Create a new branch for your feature (`git checkout -b feature/new-json-mission`). This practices the very branching and merging skills taught in Mission 6!
4. **Cultivate Your Code:** Make your changes, ensuring your code aligns with our solarpunk values—keep it readable, well-commented, and empathetic to the learner.
5. **Commit and Push:** Commit your changes with clear, descriptive messages.
6. **Open a Pull Request:** Submit your PR to the main repository. A maintainer will review your code, offering constructive feedback akin to the Code Review Mastery taught in Mission 10.

## 🤝 Community Ethos
*"Let us never stop learning from Galileo."*

When interacting in issues or pull requests, assume positive intent. We are all here to learn, ascend the ranks, and build tools that empower human resilience and knowledge sharing. Gatekeeping has no place in the Dojo.

Thank you for helping us plant these seeds!
